The Python document processor extracts text content from uploaded files using specialized libraries for each file type.
Extraction Process:
📄
Original File
➡️
📝
Extracted Text
  • PDF: PyPDF2 for text extraction
  • DOCX: python-docx for document parsing
  • PPTX: python-pptx for presentation content
# Python extraction example def extract_content(file_path): if file_path.endswith('.pdf'): return extract_pdf_text(file_path) elif file_path.endswith('.docx'): return extract_docx_text(file_path)
⬇️
3
🤖 AI Processing with LLM
Extracted content is sent to the Qwen3-coder LLM model for intelligent analysis and transformation into structured study materials.
LLM Processing:
  • Content analysis and summarization
  • Key concept identification
  • Definition extraction
  • Question generation
  • Difficulty assessment
# LLM API call response = requests.post(llm_endpoint, json={ "model": "qwen3-coder:480b-cloud", "content": extracted_text, "task": "generate_study_materials" })
⬇️

5
💾 Database Storage
Generated study materials are stored in the MySQL database with proper relationships and metadata for easy retrieval.
Database Tables:
  • notes: Study notes with content and metadata
  • flashcard_decks: Flashcard collections
  • flashcards: Individual flashcard questions and answers
  • quizzes: Quiz collections and questions
# Database insertion INSERT INTO notes (id, user_id, title, content, created_at) VALUES (uuid(), user_id, title, study_notes, file_creation_date);
⬇️
6
🖥️ Java Application Integration
The Java application retrieves the processed content and displays it in the user interface for study and review.
User Interface Features:
  • Notes management and editing
  • Flashcard study sessions
  • Quiz taking interface
  • Progress tracking
  • Search and filtering
// Java data retrieval List notes = dataStore.getNotes(); List decks = dataStore.getFlashcardDecks(); List quizzes = dataStore.getQuizzes();
